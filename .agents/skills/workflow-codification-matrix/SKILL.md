# Workflow codification matrix

Use to avoid startup bloat.

## Matrix

Match breadth, failure cost, enforcement, attention, acquisition, reuse, and independence.
Mechanisms: deterministic tool/wrapper for stable no-judgment repetition; warm recurring contextual specialist for costly repeated judgment; fresh independent specialist for adversarial judgment; disposable bounded worker for exact execution; lazy skill for conditional reference; role brief/charter for one role; enforced pipeline/tool when advice is insufficient; background wait/delegated worker for long work; tiny scoped always-on rule for high-cost invariants.
Never inject large manuals globally.

## Manager and gate

Managers route, synthesize, accept, and decide.
They work directly only when transfer plus reacquisition exceeds the bounded task, or consequential ambiguity remains after cheaper review.
Route acquisition, bulk execution, long review, artifact editing, validation loops, and waits to agents, tools, or jobs.
Return only for synthesis, acceptance, decision, completed result, or blocker.

Warm context never overrides current state.
Refresh owning source, live system, current HEAD, or artifact before state-dependent conclusions.

Advice belongs in context only when judgment must stay with the agent.
If deterministic enforcement is possible, patch the owning tool or pipeline.
Move stable judgment-free residue out of agent work; leave nuanced judgment and fallback.
Before adding startup context, prefer any narrower tool, lazy skill, role brief, background lane, or enforced gate that works.
